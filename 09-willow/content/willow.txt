
# 柳线

柳线一套中文书写格式, 模仿编程语言习惯通过换行来维护格式
句中混用英文标点, 混用英文单词, 并给英文字符增加相应空格方便阅读
加载英文标点的关键词比如 console.log , 间隔标点方便辨认
用简单直白的文字做描述, 注重文本的排版形态, 方便更快被读完

格式每一行约三十余字, 约五行形成一段, 若干段组成一块主题
主题用简单的标题概括. 这些都是为了快速阅读时定位文本
行头用一些符号来突出行与行之间的关联, 比如标题这里用了 `#`
留意文本中标点产生的空白是否能让文本显得更整齐美观

# 互联网和编程

网页的出现人们不再担心文字浪费纸张, 因而得以自由留白
我们能更快输入, 和在输入后继续调整文本的格式为阅读做优化
网络数据量巨大, 更需要斟酌文字, 调整格式, 让阅读更方便快速
同时文本内容应该直白明确, 快速被传达, 容易被记忆和遗忘

编程语言编辑器探索了大量快速输入和快速重构的方案可以被文章借鉴
限定的单行字符数, 等宽字体, 关键字高亮, 符号匹配等等
这些同样是人们能用在日常的书写和记录中用来更好表达自己的
编程同样带来大量的方案, 让人们用代码对文本的表现进行调整
未来会有更多人学会编程, 也有更多人熟悉编程的书写和编辑习惯

# 格式的尝试

柳线块状的段落突出的标题, 是为滚动和切换页面时减少丢失焦点的可能
不过现在所做还是不够让文本在移动时依然保持清晰
页面滚动时如何消除回到行头时视野定位的烦恼将被更关注
空行现在是固定的高度, 以后标题和段落组合的行距应该更多样

本文英文使用了等宽字体, 只在 Ubuntu 测试, 其他平台并不细心
为了更好地阅读, 更精神的字体, 更舒适的配色都是需要调整的
以及标题如何在不引入复杂规则的情况下能够更加醒目
Markdown 简洁的文本格式是钦佩, 并且打算借鉴和发挥的

# 一些不足和担忧

柳线的格式对行文是一种约束, 句子换行之后容易破碎, 容易简化
大概也表明了柳线只适合写说明和介绍那样浅显明确的文字
长句和规范不符, 短句几乎就变成了诗歌, 两方面都存在隐患
而且有时候人们希望把事情弄复杂, 柳线另一方面显得不那么有意义

# 未来的可能

从 Web 编程中可以借鉴更多丰富文本表现力的手法, 比如:
根据关键词甚至词性对文本进行高亮, 帮助人们发现其中的图案
对选中的字符进行匹配和高亮, 对于关键词方便掌握和调整
高亮当前阅读的区域, 方面滚动页面时更好地定位

这个页面本来可以有更多的表现力的 :(
